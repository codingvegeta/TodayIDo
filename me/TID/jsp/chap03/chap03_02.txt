JSP 페이지의 구성요소

* 디렉티브
* 스크릿트 : 스크릿트릿, 표현식, 선언부
* 표현언어
* 기본객체
* 정적인 데이터
* 표준 액션 태그
* 커스텀 태그 와 표준 태그 라이브러리

-디렉티브
JSP 페이지에 대한 설정 정보를 지정할 때 사용된다.
<%@ 디렉티브이름 속성1="값1" 속성2="값2"...%>
JSP가 제공하는 디렉티브
page : JSP 페이지에 대한 정보를 지정한다. JSP 가생성하는 문서의 타입, 출력 버퍼의 크기, 에러페이지 등등
taglib : JSP 페이지에서 사용할 태그 라이브러리를 지정한다.
include : JSP 페이지의 특정 영역에 다른 문서를 포함시킨다.

-스크립트 요소
JSP 문서의 내용을 동적으로 생성하기 위해 사용되는 것
스크립트 요소를 사용시 사용자 폼에 입력한 정보를 데이터 베이스에 저장할 수 있음
자바가 제공하는 다양한 기능 사용 가능

표현식 : 값을 출력한다.
스크립트릿 : 자바 코드를 실행한다.
선언부 : 자바 메서드를 만든다.

-기본 객체
JSP는 웹 어플리케이션 프로그래밍을 하는데 필요한 기능을 제공해주는 '기본객체'를 제공한다.
request, response, session, application, page등등

-표현 언어
JSP 스크립트 요소는 자바 문법을 그대로 사용 가능하다.
하지만 스크립트 요소를 사용하면 JSP 코드가 다소 복잡해진다.
표현언어는 '${'와'}' 사이에 정해진 문법을 따르는 식을 입렫한다.

-표준 액션 태그와 태그 라이브러리
액션태그는 JSP 에 특별한 기능을 제공한다.
액션태그는 <JSP:액션태그이름> 의 형태를 띠며 액션 태그 종류에 따라서 서로 다른 속성과 값을 갖는다.

커스텀 태그는 JSP 를 확장시켜주는 기능으로 태그형태의 기능을 제공한다.
커스텀 태그 중 자주 사용하는것들은 JSTL로 표준화 해두었다.
